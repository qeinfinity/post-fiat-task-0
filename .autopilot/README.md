# Autopilot

Autopilot is an unattended project generator + verifier.

Inputs:
- A policy-approved intake provider from `.agents/model-providers.json`
- `.autopilot/spec.md` (authoritative project spec; generated by intake)

Generated/managed:
- `.autopilot/manifest.json` (machine-readable build/test commands)
- `.autopilot/sessions/<sessionId>/` (durable logs; ignored by default)

Security:
- `.agents/security-policy.json` is enforced before generated commands run
- `.agents/agent-profiles.json` controls which agent CLIs can be launched
- `.agents/model-providers.json` controls which local/cloud intake providers can be used
- Security failures are treated as validation failures and must be fixed before readiness

Run:
- Local oMLX: start an OpenAI-compatible server, then run `bash scripts/autopilot --intake-provider=omlx-local --intake-model=<local-model>`
- OpenAI cloud: set `OPENAI_API_KEY`, then run `bash scripts/autopilot --intake-provider=openai-cloud --intake-model=<model> --allow-cloud-intake=true`
- Claude cloud: set `ANTHROPIC_API_KEY`, then run `bash scripts/autopilot --intake-provider=claude-cloud --intake-model=<model> --allow-cloud-intake=true`
